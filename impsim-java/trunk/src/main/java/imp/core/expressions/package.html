This packages contains the classes representing expressions and functions in the IMP language.

Expressions

Expressions are used to represent equations and their values. All expressions are dervied from a single abstract class, Expression. Although the imp language can use very complex and long equations, they can all be represented by only 2 classes:

  • ExpressionLiteral
  • FunctionExpression

ExpressionLiteral contains a literal value (a number, a point, a string, list of numbers, a list of points, or a list of strings). When the value of ExpressionLiterals are evaluated, they return the literal value that they hold. FunctionExpression represents calling a function with 0 or more expression arguments. This is how all operators and imp functions are evalulated. Complex expressions will have many more FunctionExpressions than ExpressionLiterals.

An example expression, 2 + 3 / 7 ^ Math~PI() would be the following in expressions

  • FunctionExpression[name="+"]
    • ExpressionLiteral[value=2]
    • FunctionExpression[name="/"]
      • ExpressionLiteral[value=3]
      • FunctionExpression[name="^"]
        • ExpressionLiteral[value=7]
        • FunctionExpression[name="PI",namespace="Math"]
Where the value of the expression is calculated by retreiving the value of each subexpression recursively.

Functions

FunctionExpressions are implemented by calling Functions that have been added to the model's FunctionManager. Functions are basically method pointers (delegates in other languages), allowing the methods to be called and registered dynamically within java.
